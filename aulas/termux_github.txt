✅ PASSO 1: Criar repositório no GitHub
   1- Acesse https://github.com
   2- Crie um novo repositório clicando em New Repository
   3- Dê um nome, marque como público ou privado, e clique em Create Repository

✅ PASSO 2: Instalar o Git no Terminal (Termux ou Linux)

#📌 Terminal no Linux:
sudo apt update                   # Atualiza a lista de pacotes disponíveis no repositório
sudo apt install git             # Instala o Git na distribuição Linux baseada em Debian (como Ubuntu)

# 📌 Terminal no Termux (Android):
pkg update                       # Atualiza os pacotes do Termux
pkg install git                  # Instala o Git no ambiente Termux

✅ PASSO 3: Verificar se o Git foi instalado com sucesso
git --version                    # Mostra a versão atual do Git instalada

✅ PASSO 3: Configurar Git com seu nome e e-mail (Obrigatório!)
git config --global user.name "Seu Nome"            # Define seu nome de usuário global
git config --global user.email "email@exemplo.com"  # Define seu e-mail usado no GitHub

✅ PASSO 4: Clonar um repositório existente do site
git clone https://github.com/usuario/repositorio.git   # Baixa o repositório para sua máquina

✅ PASSO 5: Criar uma nova pasta no repositório local
mkdir nome-da-pasta                # Cria uma nova pasta para seu projeto
cd nome-da-pasta                   # Entra na pasta do projeto
git init                           # Inicializa um novo repositório Git local (vai criar a pasta oculta .git)
                              # Envia para o GitHub

✅ PASSO 6: Adicionar arquivos ao repositório
touch nome-do-arquivo                 # Cria um arquivo de exemplo (substitua pelo seu arquivo)
git add nome-do-arquivo               # Adiciona o arquivo ao "stage", pronto para commit

#✅ PASSO 7: Realizar o primeiro commit
git commit -m "Primeiro commit: adicionando index.html"  # Cria um ponto de controle no histórico do projeto

✅ PASSO 8: Trabalhando com commits futuros
git add .                                      # Adiciona todas as alterações feitas nos arquivos
git commit -m "Descrição do que foi alterado"  # Descreve as mudanças
git push                                       # Envia os arquivos locais para o GitHub






📝 DICA FINAL:
    git status  #  Use constantemente para saber o que mudou, o que está pronto para commit, e o que ainda não foi salvo.
    git diff    # Use para ver diferenças entre arquivos modificados e os últimos commits.

✅ PASSO 9: Conectar repositório local ao GitHub
git remote add origin https://github.com/SeuUsuario/NomeDoRepositorio.git  # Substitua pela URL do seu repositório
git branch -M main                      # Renomeia o branch padrão para 'main' (caso esteja como master)
git push -u origin main                 # Envia os arquivos locais para o GitHub


✅ PASSO 10: Ver histórico de commits
git log                                 # Mostra o histórico detalhado dos commits

✅ PASSO 11: Criar e alternar branches
git branch nome-da-branch               # Cria uma nova branch (ramificação)
git checkout nome-da-branch             # Alterna para a nova branch

✅ PASSO 12: Atualizar o repositório local
git pull                                # Puxa as alterações do repositório remoto para sua máquina

✅ PASSO 13: Ignorar arquivos (opcional)
touch .gitignore                        # Cria o arquivo
echo "arquivo_secreto.txt" >> .gitignore  # Adiciona o nome do arquivo que não será rastreado pelo Git
